1

System Updates and Dependencies

First, let's update your system and install the required dependencies for Apache Spark:

sudo apt update && apt upgrade -y sudo apt install default-jdk curl -y

What this does:

  • Updates package lists and upgrades existing packages
  • Installs Java Development Kit (required for Spark)
  • Installs curl for downloading files

2

Download and Extract Apache Spark

Download Spark 3.5.6 with Hadoop 3 support and extract it to the proper location:

wget https://dlcdn.apache.org/spark/spark-3.5.6/spark-3.5.6-bin-hadoop3.tgz tar -xzvf spark-3.5.6-bin-hadoop3.tgz sudo mv spark-3.5.6-bin-hadoop3 /opt/spark

What this does:

  • Downloads the Spark binary distribution
  • Extracts the compressed file
  • Moves Spark to the standard /opt/spark directory
üì∏ Screenshot Required: Terminal showing successful download and extraction
3

Configure Environment Variables

Set up environment variables to make Spark accessible system-wide:

nano ~/.bashrc

Add these lines to the end of the file:

export SPARK_HOME=/opt/spark export PATH=$PATH:$SPARK_HOME/bin:$SPARK_HOME/sbin

Apply the changes:

source ~/.bashrc

What this does:

  • Sets SPARK_HOME to point to the installation directory
  • Adds Spark binaries to the system PATH
  • Makes Spark commands available system-wide
üì∏ Screenshot Required: ~/.bashrc file showing the added environment variables
4

Create Spark User and Set Permissions

Create a dedicated user for running Spark services (security best practice):

useradd spark chown -R spark:spark /opt/spark

What this does:

  • Creates a system user named 'spark'
  • Sets ownership of the Spark directory to the spark user
  • Enhances security by running services under a dedicated user
üì∏ Screenshot Required: Terminal showing successful user creation and permission changes
5

Configure Spark Master Service

Create a systemd service file for the Spark Master:

nano /etc/systemd/system/spark-master.service

Add the following configuration:

[Unit] Description=Apache Spark Master After=network.target [Service] Type=forking User=spark Group=spark ExecStart=/opt/spark/sbin/start-master.sh ExecStop=/opt/spark/sbin/stop-master.sh [Install] WantedBy=multi-user.target

What this does:

  • Creates a systemd service for automatic startup
  • Configures the service to run as the spark user
  • Sets up proper start and stop commands
  • Ensures the service starts after network is available
üì∏ Screenshot Required: Service file content in the editor
6

Configure Spark Slave Service

Create a systemd service file for the Spark Slave:

nano /etc/systemd/system/spark-slave.service

Add the following configuration:

[Unit] Description=Apache Spark Slave After=network.target [Service] Type=forking User=spark Group=spark ExecStart=/opt/spark/sbin/start-slave.sh spark://localhost:7077 ExecStop=/opt/spark/sbin/stop-slave.sh [Install] WantedBy=multi-user.target

Important:

Replace localhost with your actual server's IP address if setting up a distributed cluster.

What this does:

  • Creates a systemd service for the Spark worker
  • Configures the worker to connect to the master on port 7077
  • Sets up automatic startup and shutdown
üì∏ Screenshot Required: Service file with the correct configuration
7

Start and Enable Services

Reload systemd and start the services:

systemctl daemon-reload systemctl start spark-master systemctl enable spark-master systemctl start spark-slave systemctl enable spark-slave

What this does:

  • Reloads systemd to recognize new service files
  • Starts both master and slave services
  • Enables services to start automatically on boot
